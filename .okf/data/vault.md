# Location

`$HINDSIGHT_HOME`, default `~/.hindsight`. One vault, shared across every project the
user works in — this is what lets knowledge generalize across projects, not just
within one.

# Layout

```
sessions/                     one raw dump per session — see /data/session-dump.md
knowledge/
  global/<topic>.md           cross-project notes — see /data/knowledge-note.md
  global/INDEX.md             injected into every session
  projects/<proj>/<topic>.md  project-scoped notes
  projects/<proj>/INDEX.md    injected only into sessions in that project
inbox/proposals.md            skill/automation candidates — see /data/proposal-entry.md
bin/run-distill.sh            stable entrypoint the nightly schedule calls
dashboard.html                generated by /flows/dashboard.md; overwritten each run
logs/                         distill run logs + structured run history (runs.jsonl)
```

# Producers and consumers

| Location | Written by | Read by |
|---|---|---|
| `sessions/` | [/flows/capture.md](/flows/capture.md), [/flows/backfill.md](/flows/backfill.md) | [/flows/distill.md](/flows/distill.md), [/flows/reset.md](/flows/reset.md) |
| `knowledge/` | [/flows/distill.md](/flows/distill.md) | [/flows/inject.md](/flows/inject.md) |
| `inbox/proposals.md` | [/flows/distill.md](/flows/distill.md) | [/flows/proposal-approval.md](/flows/proposal-approval.md) |
| everything | — | [/flows/dashboard.md](/flows/dashboard.md) (read-only) |

# Git sync (optional)

If the vault itself is a git repo, [/flows/distill.md](/flows/distill.md) commits
after every run and pushes if an `origin` remote is configured. Without git, the same
data is still written locally — only the sync step is skipped.
